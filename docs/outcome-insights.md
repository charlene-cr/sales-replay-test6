# Outcome Insights

Outcome insights give managers a quick way to decide which calls deserve coaching time.

## How To Read The Output

- `won` calls are useful for finding repeatable discovery and close patterns.
- `lost` calls should identify the reconnect trigger before they leave the review.
- `follow-up` calls need an owner, date, and decision artifact.
- `unknown` calls should be dispositioned before they are used for coaching.

## Manager Workflow

1. Start with the highest-volume outcome for the week.
2. Compare the recommendation to the transcript notes.
3. Pick one behavior that should change in the next call.
4. Review the follow-up email or CRM note in the next one-on-one.

## Examples

- A won call may show strong discovery but weak handoff notes.
- A lost call may still contain useful objection-handling language.
- A follow-up call may be healthy if the buyer owns the next artifact.
- An unknown call should not be scored as pipeline progress.

## Review Checklist

- Confirm every outcome has at least one account example.
- Confirm the recommendation can be checked in the next manager review.
- Confirm lost calls include a reconnect trigger or explicit reason.
- Confirm follow-up calls name a date, owner, and artifact.
